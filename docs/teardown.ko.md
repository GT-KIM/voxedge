# 스냅드래곤 폰에서 완전 오프라인 음성-대-음성 AI 에이전트 돌리기 — 실제로 무엇이 필요한가

*비행기모드. 내가 말하면, 40억 파라미터 LLM이 소리 내어 답합니다 — 음성 인식, 언어 모델, 음성 합성이
전부 폰 한 대에서 로컬로, 어떤 순간에도 네트워크 없이. 첫 음성은 *인식된 텍스트* 기준 ~0.6초(LLM→TTS
경로; 말이 끝난 시점부터는 VAD 엔드포인트 + ASR이 더해짐).*

이건 제품 소개가 아니라 **teardown**입니다. *완전한* 음성 루프 — ASR → LLM → TTS, 핸즈프리, 저지연 —
이 현재 모바일 SoC에서 온전히 온디바이스로 돌아갈 수 있는지, 그리고 거기까지 가는 데 실제로 무엇이
드는지가 궁금했습니다. 아래에 아키텍처, 실기기 실측치, 며칠을 잡아먹은 버그들, 그리고 **무엇이 되고
무엇이 안 되는지** 정직한 표를 정리했습니다. 코드와 측정 하네스는 repo에 있습니다.

Apple/Google/OpenAI의 비서와 정면승부할 생각은 없습니다. 흥미롭고 비어 있는 공간은 *개발자가 직접
뜯어볼 수 있는, end-to-end 오프라인 음성 루프를 실제 실리콘에서 돌리는 것* — 그리고 그걸 되게 만드는
공학적 현실입니다.

## 아키텍처

```
mic → VAD(엔드포인트) → ASR → LLM(토큰 스트림) → ClauseSegmenter → TTS → 끊김 없는 재생
                                       └──────────────── 반이중 ────────────────┘
```

실시간처럼 느끼게 하는 핵심 아이디어 하나: **클로즈 스트리밍.** LLM 답변 전체를 기다리지도, 발화
전체에 TTS를 돌리지도 않습니다. LLM이 토큰을 흘리는 동안 세그먼터가 문장/절 경계에서 짧은 클로즈로
잘라 즉시 TTS에 넘깁니다. 클로즈 *N*을 합성하는 동안 LLM은 *N+1*을 디코드하고 스피커는 *N−1*을
재생합니다. 그래서 첫 음성은 *전체 답변*이 아니라 *첫 클로즈*에 좌우됩니다.

세 엔진은 한 앱 프로세스에 상주합니다: LLM은 NPU/HTP(Qualcomm Genie), TTS도 HTP(SNPE), ASR은
CPU(ONNX). 한 번 로드해 세션 내내 따뜻하게 유지 — 턴마다 재초기화하지 않습니다.

## 숫자 (Galaxy Z Fold7 / Snapdragon SM8750, 비행기모드)

| 단계 | 실측 |
|---|---|
| **인식된 텍스트 → 첫 음성** (LLM→TTS) | **~0.55–0.66 s** |
| LLM (Qwen3-4B, Genie w4a16, HTP) | TTFT ~65–90 ms · 디코드 ~22 tok/s · ~1.18 GB · 정상 ~70 °C |
| TTS (Supertonic short-chunk, fp16, flow 6스텝) | ~220 ms / 클로즈, 상주 |
| ASR 디코드 (SenseVoice int8 / Dolphin CTC) | ~36 ms / ~27 ms |

인앱 온디바이스 타이머로, 비행기모드에서 측정. 스펙시트나 추정이 아니라 *이 공개 프로젝트를 이 기기에서*
잰 값입니다.

## 며칠을 잡아먹은 것들 (읽을 가치가 있는 부분)

헤드라인 숫자는 진짜 작업을 가립니다. 교훈적이었던 몇 가지:

**1. NPU가 일반 앱 안에서는 안 돌았다.** 벤치(셸 바이너리)에선 HTP 백엔드가 가용했는데, 패키징된 앱
안에선 `isRuntimeAvailable = 0`에 엔진 build가 null을 반환했습니다. *CPU 라이브러리 누락이 아니었습니다.*
세 가지가 함께 맞아야 뒤집혔습니다: 매니페스트에 벤더 네이티브 라이브러리 선언(`uses-native-library
libcdsprpc.so` — Android 12+는 벤더 공개 라이브러리를 명시 요청해야 함), DSP skel 라이브러리를 런타임에
*실제* 디렉터리로 복사(FastRPC/DSP 로더는 APK 에셋 안 파일을 못 봄), 그리고 런타임 로드 *이전에*
세미콜론 구분 `ADSP_LIBRARY_PATH`(+ unsigned-PD 옵션) 구성. 어느 것도 happy-path 문서엔 없습니다.

**2. 컴파일러가 텐서를 몰래 transpose했다.** 온디바이스 그래프 컴파일러는 모든 다차원 입력의 마지막 두
축을 재배열합니다. 원래 프레임워크 레이아웃으로 먹이면 *에러 없이* 돌지만 조건화 텐서가 조용히
뒤섞입니다 — 화자 정체성·운율이 망가지고 내용은 대체로 살아남아 "그럴듯하게 틀린" 소리가 납니다. 미묘한
케이스: TTS duration predictor에 들어가는 style 텐서 하나가 잘못된 레이아웃으로 들어가 모델이 길이를
과소예측 → **모든 한국어 문장의 끝이 잘렸습니다.** 찾고 나면 수정은 사소했지만, 찾으려면 온디바이스
출력을 호스트 기준값과 숫자 단위로 대조해야 했습니다.

**3. 한국어 문장이 조용히 사라졌다.** TTS 프런트엔드가 텍스트를 NFKD 정규화하는데, 한글 음절을 자모
2~3개로 분해합니다. 모델 입력 shape는 고정 64토큰. 25자처럼 보이던 클로즈가 분해 후 ~70토큰이 되어
shape를 넘기고 예외를 던졌고 — 파이프라인이 그걸 잡아서 *건너뛰었습니다.* 결과: 긴 한국어 클로즈가
에러 표시도 없이 음성 출력에서 가끔 사라졌습니다. 수정: 클로즈를 원시 글자수가 아니라 *NFKD 토큰
길이*로 자르기. (영어는 분해가 없어 한 번도 안 걸림.)

**4. 측정이 내 계획을 두 번 반증했다.** 배경음악이 ASR을 망칩니다. 첫 직감은 "더 강한 denoiser를 쓰자".
CER 하네스를 만들고 *실제* 잡음 발화를 실제 마이크 경로로 캡처해 측정했더니, 더 큰 신경망 denoiser는
한국어를 **더 나쁘게** 만들었고(종성을 손상), 작은 건 도움이 전혀 안 됐습니다. 진짜 해법은 ASR *모델*을
언어별로 바꾸는 것 — 한국어 튜닝 CTC 모델이 그 클립의 CER을 0.60→0.30으로 낮추고 더 빨랐습니다 —
그리고 **denoiser를 아예 빼는 것**이었습니다. 제 초기 베팅 둘 다 틀렸고, 출하 전에 그걸 알려준 건
하네스뿐이었습니다. 지금 repo에서 가장 가치 있는 게 그 하네스입니다.

## 되는 것, 안 되는 것 (정직한 부분)

| 능력 | 상태 |
|---|---|
| 완전 오프라인(비행기모드) 한/영 음성 루프 | ✅ 동작, 측정됨 |
| 핸즈프리 VAD 턴테이킹 | ✅ 동작 |
| ~0.6초 첫 음성 (클로즈 스트리밍) | ✅ SM8750 실측 |
| ASR + 4B LLM + TTS 한 프로세스 공존 | ✅ 동작 |
| 배경음악 하 ASR | ⚠️ 개선(언어별 모델), 미해결 |
| LLM 답변 품질 | ⚠️ 4B급; 프롬프트·history 튜닝, GPT급 아님 |
| Barge-in(답변 중 끼어들기) | 🧪 실험적, 기본 OFF |
| iOS | ❌ 미구현 (로드맵) |
| SoC 간 이식성 | ❌ 단일 SoC(SM8750); SoC마다 graph-prep 필요 |

둘은 부연이 필요합니다. **Barge-in** — 어시스턴트가 말하는 중 사용자가 끼어들기 — 은 진짜 어려웠습니다:
재생 중 마이크를 열어두고 플랫폼 AEC를 켜도 잔여가 남아 어시스턴트가 *자기 자신을* 끊었습니다. 실험
플래그 뒤 기본 OFF로 두고, 실제로 잘 되는 깨끗한 반이중 분리(턴 동안 마이크 음소거 + 짧은 tail guard)를
씁니다. **이식성**은 이 부류 작업의 정직한 천장입니다: 모델은 특정 SoC용으로 컴파일·graph-prep되어,
다른 스냅드래곤 — 하물며 다른 벤더 — 로 옮기는 건 재컴파일이 아니라 기기별 실작업입니다.

## 다음에 시도할 것

더 나은 far-field 강건성(denoiser보다 캡처 경로·오디오 소스가 더 중요), barge-in을 믿을 수 있게 하는
참조신호 기반 에코 제거, 크로스플랫폼 검증을 위한 Core ML / MLX iOS 경로, 그리고 한 칩에 용접되지 않게
할 이식성 레이어.

## 마무리

이 프로젝트의 목적은 완성된 제품이 아니라, 오늘의 모바일 실리콘에서 완전 오프라인 음성 루프가 *실제로
얼마나 드는지*를 측정한, 정직한 레퍼런스입니다 — 지연 숫자가 가리는 전쟁담과 함께. 온디바이스/엣지 AI를
하신다면 봐주시면 기쁩니다: 코드·숫자·한계가 전부 repo에 있고, 이슈·비판 환영합니다.
